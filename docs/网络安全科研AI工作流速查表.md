## 阶段间主要回退路径

```
1 ⇄ 2 ⇄ 3 ⇄ 4.1 → 4.2 ⇄ 5 → 6 ⇄ 7
↑    ↑     ↑      ↑      ↑      ↑
│    │     │      │      │      │
│    └─ 缺口已被填 ─→ 换方向  │      │
│          │      │      │      │
│          └─ 方案不可行 ─┘      │      │
│                 │             │      │
│                 └─ 实验不足 ──┘      │
│                        │            │
│                        └─ 逻辑漏洞 ─→ 补实验
│                               │
└─── 审稿要求补实验 ←────────────┘
```

- **阶段二 → 阶段一**：文献调研发现研究问题已被解决
- **阶段三 → 阶段二**：方案设计时发现技术细节调研不足
- **阶段四.1 → 阶段三**：原型验证发现方案不可行
- **阶段四.2 → 阶段三**：大规模实验系统性差于基线
- **阶段五 → 阶段三/四.2**：审稿发现方法论缺陷或实验不足
- **阶段六 → 阶段四.2**：写作时发现实验数据不支撑核心论证
- **阶段七 → 阶段四.2**：审稿人要求补充实验

## 核心原则

- **AI 为副驾驶，核心研究判断人工主导**
- **代码实现、数据处理、代码审查由 AI 原生能力完成，无需专用 Skill**
- **每阶段结束执行 `/neat` 归档，保障项目可回溯**
- **仅用 `unslop` 做语言优化，禁止用于规避 AI 检测**
- **迭代是常态——在审稿前自己发现问题远比被拒稿后补救高效**
